# 积木平衡游戏 - 产品需求文档

## 1. 产品概述

一款极简风格的物理平衡小游戏，玩家需要从积木塔底部小心翼翼地抽取积木，保持塔身平衡不倒即为胜利。游戏追求"一秒上手，十分钟上瘾"的体验，以清新明快的视觉风格和真实的物理模拟为核心卖点。

**核心理念**：简单到一句话就能说明白，但玩起来却充满挑战和乐趣。

## 2. 核心功能

### 2.1 游戏页面模块

| 模块 | 功能描述 |
|------|----------|
| 积木塔 | 由多层积木组成的塔状结构，每层3块积木交错堆叠 |
| 抽取区域 | 高亮显示可抽取的积木，支持鼠标拖拽操作 |
| 计分系统 | 记录成功抽取的积木数量 |
| 重置按钮 | 一键重新开始游戏 |
| 游戏状态 | 显示"游戏中"、"成功"、"失败"三种状态 |

### 2.2 积木系统

- **积木尺寸**：标准统一尺寸，层与层之间交错堆叠
- **颜色区分**：每层积木采用不同色调，确保视觉清晰可辨
- **纹理处理**：每种颜色配不同纹理图案（条纹、点状、网格、纯色等）
- **物理属性**：质量、摩擦力、稳定性等参数

### 2.3 游戏规则

1. 点击或拖拽底部积木进行抽取
2. 积木必须完全移出塔身范围才算成功
3. 抽取后塔身倾斜超过45度判定为失败
4. 成功抽取所有积木为终极胜利

## 3. 用户交互流程

```
开始游戏
    ↓
积木塔生成（随机颜色分配）
    ↓
玩家选取底部可抽取积木（鼠标悬停高亮）
    ↓
拖拽积木向外移动
    ↓
判断：是否完全移出塔身？
    ├── 否 → 积木回归原位
    └── 是 → 检查塔身稳定性
              ↓
        判断：塔身倾斜角度 < 45°？
            ├── 否 → 游戏结束（失败）
            └── 是 → 计分 + 生成下一层可抽取积木
                      ↓
                  判断：还有积木可抽？
                      ├── 否 → 胜利
                      └── 是 → 继续抽取循环
```

## 4. 视觉设计

### 4.1 整体风格
- **美学方向**：清新糖果风 + 几何极简主义
- **背景**：渐变天空蓝至浅粉色，营造温暖愉悦氛围
- **质感**：积木带有轻微圆角和柔和阴影，呈现立体感

### 4.2 色彩方案

| 积木类型 | 主色 | 纹理 |
|----------|------|------|
| 红色系 | #FF6B6B | 斜条纹 |
| 橙色系 | #FFA94D | 点状 |
| 黄色系 | #FFE066 | 方格 |
| 绿色系 | #69DB7C | 横纹 |
| 青色系 | #38D9A9 | 波浪纹 |
| 蓝色系 | #4DABF7 | 纯色+描边 |
| 紫色系 | #B197FC | 圆点 |

### 4.3 字体
- **标题**：Baloo 2（圆润可爱）
- **数字/分数**：Orbitron（几何感）

### 4.4 动效设计
- 积木悬停：轻微上浮 + 发光边框
- 抽取成功：粒子庆祝效果
- 游戏失败：积木散落动画
- 胜利：彩虹渐变 + 星星特效

## 5. 技术实现

### 5.1 技术栈
- **物理引擎**：Matter.js（轻量级 2D 物理引擎）
- **渲染**：HTML5 Canvas
- **构建工具**：Vite
- **框架**：原生 JavaScript（单文件，无依赖）

### 5.2 物理参数
- 重力：标准地球重力
- 积木摩擦力：0.8（高摩擦防止滑落）
- 积木弹性：0.1（低弹性保持稳定）
- 倾斜阈值：45度

- [ ] 拖拽交互流畅自然
- [ ] 物理模拟真实可信
- [ ] 倾斜超过45度能正确判定失败
- [ ] 计分系统正常工作
- [ ] 重置功能正常
- [ ] 各积木颜色纹理清晰可辨
- [ ] 视觉风格清爽养眼
