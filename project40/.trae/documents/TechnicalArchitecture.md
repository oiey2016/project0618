# 积木平衡游戏 - 技术架构文档

## 1. 架构设计

```
┌─────────────────────────────────────────────────────┐
│                    游戏主页面                        │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │
│  │  游戏画布   │  │   积分显示   │  │  控制按钮   │  │
│  │  (Canvas)   │  │  (HTML)    │  │  (HTML)    │  │
│  └─────────────┘  └─────────────┘  └─────────────┘  │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│              Matter.js 物理引擎层                   │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐      │
│  │ Bodies │ │ World  │ │ Engine │ │ Events │      │
│  └────────┘ └────────┘ └────────┘ └────────┘      │
└─────────────────────────────────────────────────────┘
```

## 2. 技术选型

| 技术 | 用途 | 版本 |
|------|------|------|
| Matter.js | 2D物理引擎 | 0.19.0 |
| HTML5 Canvas | 渲染积木和背景 | - |
| 原生 JavaScript | 游戏逻辑 | ES6+ |

## 3. 模块划分

### 3.1 核心模块

| 模块 | 职责 |
|------|------|
| `Game` | 游戏主控制器，管理游戏状态和生命周期 |
| `Tower` | 积木塔生成器，负责创建和管理积木结构 |
| `Block` | 单个积木类，管理积木的外观、颜色、纹理 |
| `Physics` | Matter.js 封装，提供物理模拟能力 |
| `InputHandler` | 输入处理，鼠标拖拽交互 |
| `Renderer` | 自定义Canvas渲染，绘制纹理和特效 |
| `UI` | 积分显示、游戏状态提示、重置按钮 |

### 3.2 文件结构

```
index.html          - 入口页面
├── 内联 CSS         - 样式定义
└── 内联 JavaScript  - 游戏逻辑（单文件实现）
```

## 4. 积木数据结构

```javascript
// 积木属性
{
  id: number,           // 唯一标识
  x: number,            // X坐标
  y: number,            // Y坐标
  width: number,        // 宽度 (100)
  height: number,       // 高度 (40)
  angle: number,        // 旋转角度
  color: string,        // 主色调
  texture: string,      // 纹理类型
  isExtractable: bool,  // 是否可抽取
  layer: number         // 所在层级
}

// 塔结构
{
  layers: number,        // 总层数 (10)
  blocks: Block[],       // 所有积木
  currentExtractable: Block[]  // 当前可抽取积木
}
```

## 5. 物理引擎配置

```javascript
{
  gravity: { x: 0, y: 1 },
  blockOptions: {
    friction: 0.8,       // 高摩擦防止滑落
    frictionStatic: 1.0, // 静摩擦
    restitution: 0.1,    // 低弹性
    density: 0.001       // 密度
  },
  tiltThreshold: 45      // 倾斜失败阈值（度）
}
```

## 6. 渲染管线

```
游戏循环 (60fps)
    │
    ▼
更新物理引擎 (Matter.Engine.update)
    │
    ▼
检查塔身倾斜角度
    │
    ▼
渲染背景渐变
    │
    ▼
渲染所有积木
    │
    ├── 绘制主体颜色
    ├── 绘制纹理图案
    └── 绘制阴影效果
    │
    ▼
渲染UI层（积分、状态）
    │
    ▼
渲染特效层（粒子、庆祝）
```

## 7. 纹理绘制策略

使用 Canvas 2D API 绑定材质：

| 纹理类型 | 绘制方法 |
|----------|----------|
| 斜条纹 | `ctx.lineTo()` 循环绘制 |
| 点状 | `ctx.arc()` 循环绘制 |
| 方格 | `ctx.strokeRect()` 网格 |
| 横纹 | `ctx.fillRect()` 水平线条 |
| 波浪纹 | `ctx.quadraticCurveTo()` |
| 纯色 | 无纹理，仅边框 |

## 8. 交互流程

```
鼠标悬停
    ↓
检测悬停位置是否有可抽取积木
    │
    ▼
是 → 高亮该积木（发光边框 + 上浮效果）
    │
    ▼
鼠标按下
    │
    ▼
记录初始位置，开始拖拽
    │
    ▼
鼠标移动 → 更新积木位置（物理引擎同步）
    │
    ▼
鼠标释放
    │
    ▼
判断积木是否完全移出塔身区域
    │
    ├── 否 → 积木弹回原位
    │
    └── 是 → 触发物理模拟
              ↓
          检查塔身倾斜角度
              │
              ├── >= 45° → 游戏失败
              │
              └── < 45° → 计分 + 更新可抽取积木
```

## 9. 性能优化

- 使用 `requestAnimationFrame` 保持60fps
- 物理引擎更新与渲染分离
- 纹理预渲染到离屏Canvas
- 积木状态变化才触发重绘
